# Technical Details

## What it does

The QR code generator is a single HTML file (~200 lines) that encodes arbitrary text or WiFi credentials into a QR code rendered on a `<canvas>` element. It uses the `qrcode-generator` library (kazuhikoarase, v1.4.4) loaded from jsDelivr CDN to perform Reed-Solomon error correction encoding and module layout. Everything runs client-side — no data leaves the browser.

WiFi QR codes follow the `WIFI:T:<security>;S:<ssid>;P:<password>;H:<hidden>;;` format recognized by iOS and Android camera apps. Special characters (`;`, `,`, `:`, `\`, `"`) in SSID/password fields are backslash-escaped per spec.

## Architecture

```
qr-code-generator.html  (single file, all-in-one)
├── <style>       — Responsive CSS, segmented control, form styling
├── HTML          — Two mode panels (text / WiFi), style options, canvas output
└── <script>
    ├── switchMode()       — Toggle between URL/Text and WiFi input panels
    ├── buildWifiString()  — Encode WiFi credentials into WIFI: URI format
    ├── generate()         — Create QR via library, render to canvas with options
    └── downloadQR()       — Export canvas as PNG via toDataURL
```

**Data flow:**
1. User enters text or WiFi credentials
2. `generate()` builds the input string (plain text or WiFi URI)
3. `qrcode(0, 'M')` auto-selects the minimum QR version for the data length, with Medium error correction
4. Modules are drawn to a `<canvas>` with configurable cell size, color, rounded corners, and quiet zone
5. Download button exports the canvas as `qrcode.png`

**Dependencies:**
- `qrcode-generator@1.4.4` via CDN (7 KB minified) — the only external dependency
- Python 3 `http.server` in `run.sh` (optional, for local serving)

## Style options

| Option | Values | Effect |
|---|---|---|
| Module style | Square / Rounded | Rounded uses `roundRect` with 35% corner radius |
| Size | Small (4px) / Medium (6px) / Large (10px) | Cell size in pixels per module |
| Foreground color | Any hex color | Applied to dark modules; background is always white |
| Quiet zone | On / Off | Adds 4-module white border around QR code (improves scan reliability) |

## Limitations

- **No logo/image overlay** — adding a center logo requires higher error correction and masking logic not included here.
- **No SVG export** — output is PNG only via canvas.
- **CDN dependency** — the qrcode-generator library is loaded from jsDelivr. For fully offline use, inline the library or bundle it.
- **No QR decoding** — this tool generates codes only; it cannot read/scan existing QR codes.
- **Canvas `roundRect` support** — rounded module style requires a browser that supports `CanvasRenderingContext2D.roundRect()` (Chrome 99+, Firefox 112+, Safari 15.4+).

## Why it matters for Claude-driven products

- **Lead generation / marketing:** Instantly create QR codes linking to landing pages, signup forms, or promo URLs. A Claude skill can generate these on-the-fly during content creation workflows.
- **Event & retail:** WiFi QR codes for guest networks at offices, cafes, or events — a common small-business ask that Claude can handle in one turn.
- **Agent factories:** This pattern — a single self-contained HTML tool generated by a skill — is a template for building disposable micro-tools. The skill teaches Claude to produce complete, working artifacts without npm installs or build steps.
- **Ad creatives:** QR codes on print ads, flyers, or business cards that link to tracked URLs. Pair with a URL shortener skill for full funnel tracking.
