# Mobile & Remote Codex Workflow

Manage, monitor, steer, and approve AI-assisted coding tasks in real time from any device — including mobile — using OpenAI Codex via the ChatGPT app.

## When to use

- "I want to monitor my Codex coding tasks from my phone"
- "How do I approve or steer Codex changes remotely?"
- "Set up a mobile-friendly remote coding workflow with AI"
- "I need to manage coding agents across multiple devices"
- "Help me review and approve AI-generated code on the go"

## How to use

### 1. Set up Codex access on mobile

- Install the **ChatGPT mobile app** (iOS or Android)
- Sign in with your OpenAI account that has Codex access
- Navigate to the Codex interface within the app

### 2. Start or connect to a coding task

- Open an existing Codex coding session or create a new one
- Describe your coding task in natural language (e.g., "Add input validation to the signup form" or "Refactor the database connection module")
- Codex will begin working on the task in a cloud-based environment

### 3. Monitor progress in real time

- Watch Codex as it reads your codebase, plans changes, and writes code
- View file diffs and proposed modifications as they are generated
- Track task status across multiple concurrent coding tasks

### 4. Steer and guide the agent

- Provide mid-task feedback: "Focus on the error handling first" or "Use async/await instead of callbacks"
- Redirect the agent if it goes off track
- Add constraints or clarifications without restarting the task

### 5. Review and approve changes

- Review generated diffs and code changes from your device
- Approve changes to merge them into your codebase
- Request revisions or reject changes that don't meet requirements
- Create pull requests directly from the Codex interface

### Best practices

- **Write clear task descriptions** — specificity reduces back-and-forth steering
- **Break large tasks into smaller units** — easier to review and approve on mobile
- **Use steering prompts early** — guide the agent before it goes too far in the wrong direction
- **Review diffs carefully before approving** — especially for security-sensitive or production code
- **Leverage cross-device continuity** — start a task on desktop, monitor and approve from mobile
