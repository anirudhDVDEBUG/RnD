# Technical Details вҖ” Cord Agent Fabric

## What it does

Cord is a **decentralized service-discovery fabric** purpose-built for AI infrastructure. It lets LLM CLI tools, HTTP model backends, and MCP servers register themselves with natural-language descriptions, then makes them discoverable across machines via semantic search queries. Instead of maintaining a central registry or hard-coding endpoints, agents join a peer mesh and find each other by *describing what they need*.

The real Cord is written in Rust for low latency and minimal resource usage. This prototype contains a Python simulation (`cord_demo.py`) that replicates the core registration вҶ’ discovery вҶ’ mesh loop using TF-IDF cosine similarity, demonstrating the concept without requiring the Rust toolchain or a multi-machine network.

- **Registration**: Each node runs a Cord daemon. Services register with a name, type, endpoint, and free-text description.
- **Mesh networking**: Peers discover each other via a gossip protocol вҖ” no central coordinator.
- **Semantic search**: Queries are matched against service descriptions using embedding similarity. The real Cord likely uses a local embedding model; our simulation uses TF-IDF.
- **Multi-protocol**: Supports MCP servers (stdio/SSE), HTTP LLM backends, and CLI-based agents.

### This prototype (Python)

```
cord_demo.py
в”ңв”Җв”Җ CordFabric          вҖ” In-memory registry + peer list
в”ңв”Җв”Җ register()          вҖ” Add a service with metadata
в”ңв”Җв”Җ add_peer()          вҖ” Simulate joining a mesh peer
в”ңв”Җв”Җ discover(query)     вҖ” TF-IDF cosine similarity search
в””в”Җв”Җ run_demo()          вҖ” End-to-end scenario with 8 services, 5 peers
```

**Key files:**

| File | Purpose |
|------|---------|
| `cord_demo.py` | Core simulator вҖ” registration, discovery, mesh, demo |
| `run.sh` | One-command launcher |
| `cord_mesh_state.json` | Exported mesh state (generated by run.sh) |
| `requirements.txt` | Dependencies (stdlib only) |

**Data flow:**

1. Services register with `CordFabric.register()` вҖ” stored in an in-memory list.
2. Peers join with `add_peer()` вҖ” each peer tracks which services it hosts.
3. `discover(query)` tokenizes the query, computes TF-IDF vectors for all service descriptions, and returns the top-k by cosine similarity.
4. Results and full mesh state are printed and exported as JSON.

**Dependencies:** Python 3.7+ standard library only (json, math, re, dataclasses, collections).

## Limitations

- **Simulation only**: This prototype does not open network sockets or form a real mesh. It demonstrates the *concept* of semantic discovery, not the actual gossip protocol.
- **TF-IDF vs. embeddings**: Real Cord presumably uses dense embeddings (e.g., a small local model). TF-IDF works for demos but lacks understanding of synonyms and paraphrases.
- **No persistence**: The in-memory registry is ephemeral. Real Cord likely persists state across restarts.
- **No authentication**: The demo has no access control. A production mesh would need peer authentication and service-level authorization.
- **Early-stage project**: Cord is relatively new. The CLI interface and config format may change. Check the repo for current docs.

## Why this matters for Claude-driven products

### Agent factories
Cord solves the "how do my agents find each other" problem. If you're building an agent factory that spawns specialized agents (research, code, deploy), Cord gives them a discovery layer вҖ” each agent registers what it does and finds collaborators by description, not by hard-coded URLs. This enables dynamic, elastic multi-agent architectures.

### MCP server orchestration
As the MCP ecosystem grows, teams will run dozens of MCP servers across machines. Cord turns "which server handles git?" from a config-file lookup into a semantic query. This is especially useful for Claude Code workflows where new MCP servers are added frequently.

### Lead-gen and marketing pipelines
Multi-step pipelines (scrape вҶ’ enrich вҶ’ score вҶ’ personalize) often run across different nodes. Cord lets each pipeline stage advertise its capabilities and discover upstream/downstream services automatically вҖ” no manual wiring.

### Voice AI
Distributed voice pipelines (ASR on edge, LLM in cloud, TTS on edge) benefit from dynamic discovery. If a GPU node goes down, Cord's mesh lets the pipeline find an alternative backend without reconfiguration.
