# Technical Details

## What it does

`humanize_chinese` is a rule-based Chinese text post-processor that targets the most recognizable patterns of AI-generated Chinese. It maintains a curated table of ~20 formal-to-colloquial connector swaps, removes common hedging openers, naturalizes stiff verb phrases, varies sentence rhythm by splitting overly long enumeration sentences, and optionally inserts discourse markers to mimic natural spoken flow. The output preserves the original meaning while shifting the register from "machine essay" to "human conversation."

This is **not** an LLM wrapper — it uses zero API calls. The entire transformation is deterministic regex + string replacement with a seeded RNG for randomized variant selection. This makes it fast (<1ms per paragraph), free, and fully offline.

## Architecture

```
humanize_chinese.py          (single file, ~180 LOC)
├── Replacement tables       (FORMAL_TO_COLLOQUIAL, HEDGING_PHRASES, VERB_SWAPS, DISCOURSE_MARKERS)
├── humanize()               (core function — applies rules in order, returns HumanizeResult)
└── main()                   (CLI entry point — argparse, file/stdin/arg input)
```

### Data flow

1. **Input**: raw Chinese text string + optional register (`casual`/`professional`/`creative`)
2. **Pass 1**: Formal connector → colloquial swap (e.g., 此外 → 另外)
3. **Pass 2**: Remove hedging sentence openers (e.g., 总的来说，)
4. **Pass 3**: Stiff verb phrase → natural alternative (e.g., 非常好 → 挺好的)
5. **Pass 4**: Sentence-length variation — split long sentences with enumeration lists
6. **Pass 5**: Insert discourse markers before second sentence (casual register only)
7. **Pass 6**: Punctuation cleanup
8. **Output**: `HumanizeResult(original, humanized, changes)`

### Dependencies

- **Python 3.10+** (uses `X | Y` union syntax)
- **Zero external packages** — stdlib only (`re`, `random`, `dataclasses`, `argparse`)

## Limitations

- **Rule-based only**: Handles the top ~20 AI-Chinese patterns but cannot catch every instance of robotic phrasing. A fine-tuned model would cover more cases.
- **No semantic understanding**: It swaps patterns mechanically. Rarely, a swap may subtly shift connotation (e.g., 然而 in a literary context should stay 然而).
- **Single-pass replacements**: Each pattern is replaced at most once. If the same formal connector appears 3 times, only the first is swapped.
- **No paragraph-level restructuring**: It works sentence-by-sentence. It won't reorder paragraphs or merge/split sections.
- **Casual register only for discourse markers**: Professional and creative registers skip the discourse-marker insertion step, which is the most "humanizing" transform.
- **Simplified Chinese only**: Does not handle Traditional Chinese (繁體字) variants.

## Why this matters for Claude-driven products

| Use case | Application |
|---|---|
| **Marketing / ad creatives** | Chinese ad copy generated by Claude often reads stiff. Pipe it through `humanize()` as a post-processing step before publishing to Weibo, WeChat, or Xiaohongshu. |
| **Lead-gen chatbots** | Prospects in Chinese-speaking markets disengage from robotic responses. Use the skill to make Claude's replies feel natural in real-time conversations. |
| **Agent factories** | If you're building multi-agent systems that generate Chinese content at scale, this is a cheap deterministic filter that catches the worst AI tells before human review. |
| **Content localization** | Teams translating English Claude output into Chinese can run this as a final polish step to eliminate translationese. |

The skill version (SKILL.md) is especially useful: drop it into `~/.claude/skills/` and Claude Code itself applies these rules without any code — just say "humanize this Chinese text."
